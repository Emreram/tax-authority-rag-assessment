================================================================================
GENERATION SYSTEM PROMPT — Answer Generation with Mandatory Citations
================================================================================

PURPOSE:
  Generate a legally grounded answer using ONLY the provided context passages.
  Every factual claim must include an inline citation referencing the specific
  chunk that supports it. No prior knowledge, no guessing, no extrapolation.

WHEN USED:
  After retrieval grading confirms sufficient RELEVANT context (>=3 chunks).
  Called by: module3_crag_statemachine.py → generate() node
  Temperature: 0.0 (deterministic output — no creativity needed for legal facts)

CITATION FORMAT:
  [Source: {chunk_id} | {hierarchy_path}]
  Example: [Source: WetIB2001-2024::art3.114::lid1::chunk001 | AWR > Art 3.114 > Lid 1]

WHAT HAPPENS IF CITATION RULES ARE VIOLATED:
  The validate_output() node (next step in the CRAG state machine) performs
  a set-membership check: every cited chunk_id must exist in the graded context.
  If any citation references a chunk_id NOT in the context → response is REJECTED.
  If the response contains NO citations at all → response is REJECTED.
  Rejected responses are replaced by a polite refusal (refuse() node).

DESIGN NOTES:
  - "ONLY use information from the provided context" prevents the LLM from
    injecting its training knowledge. Critical for legal accuracy — the LLM's
    parametric knowledge may be outdated or jurisdictionally wrong.
  - The context block includes full metadata (hierarchy_path, article_num, etc.)
    so the LLM has the raw material to construct proper legal citations.
  - "Admit gaps rather than guessing" aligns with Assumption A16: prefer false
    negatives over false positives.
